# growi-plugin-gantt

Redmine 風の**チケット一覧**と**ガントチャート**を GROWI で使うためのスクリプトプラグインです。`/tickets` 配下のページをチケットとして扱い、本文の `ticket-meta` ブロックで状態・担当・開始日・期日・進捗を管理します。

## 機能

- **チケット一覧**: `/tickets` 配下のページを一覧表示。状態（New / In Progress / Done）の変更を保存。
- **ガントチャート**: 開始日・期日が設定されたチケットを [Frappe Gantt](https://github.com/frappe/gantt) で表示。バーをクリックするとポップアップから「ページを開く」で該当ページへ遷移。
- **メタ編集**: 一覧で状態を変更すると、該当ページ本文の `ticket-meta` ブロックが API で更新されます。

## 必要な環境

- GROWI（オンプレ / GROWI.cloud）
- ブラウザから `/_api/v3/pages/list` および `/_api/v3/page`、`/_api/v3/pages/:id`（PUT）が同一オリジンで利用できること

## インストール

1. このリポジトリをビルドし、`dist/` を配信できる URL を用意する（後述の「ビルド」を参照）。
2. GROWI の**管理画面** → **プラグイン** を開く。
3. **スクリプトプラグイン**の「プラグインの URL」に、`dist/` の **manifest.json や JS/CSS が読める URL** を指定（例: リポジトリの raw や GitHub Pages のルートを `dist` にした場合など）。
4. **インストール** → **有効にする**。
5. サイドバーに **「📊 チケット・ガント」** ボタンが表示されます。

**サイドバーにボタンが出ない場合**

- 画面左端に **「📊 チケット・ガント」** のフォールバックボタンが出ていないか確認してください。
- ページを再読み込みしてから再度試してください。

## 使い方

1. **パネルを開く**: サイドバーの **「📊 チケット・ガント」** をクリック。
2. **タブ**: 「🎫 チケット」で一覧、「📊 ガント」でガントチャートを表示。
3. **チケット一覧**: タイトルはページへのリンク。状態のドロップダウンを変更すると、そのページの本文メタが更新されます。
4. **新規チケット**: 「＋ 新規チケット」リンクで `/tickets/new` を開き、そこでページを作成。本文の先頭に `ticket-meta` ブロックを追加すると一覧・ガントに表示されます。
5. **ガント**: 開始日・期日が両方あるチケットだけが表示されます。バーをクリック → ポップアップの「ページを開く」で該当ページを開けます。
6. **サンプル**: すぐに試す場合は [sample-tickets.md](./sample-tickets.md) にサンプルチケットの本文を掲載しています。`/tickets` 配下に新規ページを作成し、コピー＆ペーストして保存してください。

## リンクについて

- GROWI v5 以降は **パーマリンク（ID形式）** が標準です（例: `https://example.com/69b6b1a9060f0eb374bcc7e6`）。ページ名を変更してもリンクが変わらないため、チケット一覧・ガントからのリンクは **pageId があるときはパーマリンク** を使います。
- pageId がない場合（例: 新規チケット用の `/tickets/new`）は **パス形式** の URL を使用します。
- 参考: [パスや URL をクリップボードにコピーする | GROWI Docs](https://docs.growi.org/ja/guide/features/copy_to_clipboard.html)

## チケットのメタ（ticket-meta）

各チケットページの本文**先頭**に、次のようなコードブロックを置きます。

````
```ticket-meta
status: New
project: ""
assignee: ""
startDate: ""
dueDate: ""
progress: 0
```
````

| キー | 説明 |
|------|------|
| `status` | 状態: `New` / `In Progress` / `Done` |
| `project` | プロジェクト名（任意文字列） |
| `assignee` | 担当者（任意文字列） |
| `startDate` | 開始日（YYYY-MM-DD） |
| `dueDate` | 期日（YYYY-MM-DD） |
| `progress` | 進捗 0～100（数値） |

- ブロックの形式は **YAML**（`key: value` の並び）です。従来の JSON 形式でも読み取りは対応していますが、プラグインで保存する際は YAML で書き出します。
- ガントに表示されるのは `startDate` と `dueDate` が両方設定されていて、かつ開始日 ≦ 期日であるチケットです。
- プラグインの一覧で「状態」を変更すると、このブロックの `status` が API で書き換えられます。

## ビルド

```bash
npm ci
npm run build
```

- 成果物は `dist/` に出力されます。`dist/assets/` に JS と CSS、`dist/.vite/manifest.json` が生成されます。
- GROWI のスクリプトプラグインは、この `dist/` の内容が配信される URL を「プラグインの URL」に指定して利用します。
