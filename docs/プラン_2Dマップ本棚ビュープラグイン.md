# プラン: 2D マップ風・本棚ビュー スクリプトプラグイン

GROWI のページツリーを「2D マップ」と「本棚ビュー」で表示し、ナビ・探索の体験を向上させるスクリプトプラグインの計画書。  
参考: Knowledge_Library の `explore-2d` / `shelf`。

---

## 1. 目標

- **ツリーの別表現** … 従来のサイドバーツリーに加え、「廊下＋棚」の 2D マップと「カード一覧」の本棚ビューを提供する。
- **GROWI 内で完結** … データは GROWI API のみ。認証・ページ取得は GROWI に任せる。
- **スクリプトプラグイン** … クライアント側のみで実装し、GROWI の既存 UI に組み込む。

---

## 2. 機能概要

| 機能 | 説明 | 表示場所案 |
|------|------|------------|
| **2D マップ** | 現在パス配下の「子」を、奥に向かって並んだ棚として表示。棚クリックでそのパスへ遷移。一点透視の廊下風。 | サイドパネル or モーダル or 専用タブ |
| **本棚ビュー** | あるパスを「棚」とみなし、その直下のページをカード（表紙・タイトル・ページ数など）で一覧。クリックでページを開く。 | 同上 |

---

## 3. 参照元（Knowledge_Library）の整理

### 3.1 2D マップ（explore-2d）

- **データ**: 「本」（＝ページまたはパス）を分野ごとに分類し、棚（shelf）に割り当て。
- **UI**: 一点透視の廊下。手前から奥に複数行の棚。各棚は「分野名 + 冊数」。クリックでその分野の本一覧へ。
- **GROWI への置き換え**: 「分野」= 直下の子パス（またはパスの先頭セグメント）。「本」= その配下のページ。GROWI には「分野」がないので、**直下の子ディレクトリ名を棚ラベル**にする。

### 3.2 本棚（shelf）

- **データ**: ある room（パス）の level（階層）に属する「本」一覧。
- **UI**: カード一覧。各カード: 表紙風、タイトル、説明、ページ数。クリックで本の詳細（目次）へ。
- **GROWI への置き換え**: 「棚」= あるパス（例: `/project/設計`）。「本」= その直下の子ページ or 子パス。カードクリックで GROWI のそのページを開く。

---

## 4. データ取得（GROWI API）

- **使用する API（想定）**
  - ページ一覧: `GET /_api/v3/pages/list?path={path}` で指定パス配下のページを取得。
  - 必要に応じて `GET /_api/v3/page` で単一ページの詳細（要約など）を取得。
- **認証**: ブラウザで GROWI にログイン済みであれば、同一オリジンで API を呼べる。トークンは GROWI のセッションに含まれる想定（要確認: クライアントから API を叩く方法）。
- **フォールバック**: API が使えない場合は、現在ページの DOM や `growiFacade` から取得できる情報（現在パス、サイドバーツリーのノード）だけでも「現在のページの兄弟・子」を簡易表示する方法を検討する。

---

## 5. UI 設計

### 5.1 起動方法

- **案 A**: サイドバーに「本棚」「2D マップ」ボタンを追加し、クリックでパネルを開く。
- **案 B**: ページ上部のツールバーや「…」メニューに「本棚で見る」「マップで見る」を追加。
- **案 C**: 特定の URL クエリ（例: `?view=shelf`）やハッシュで表示を切り替え。

→ **まずは案 A（サイドバーにボタン＋パネル）** で進め、必要なら B/C を追加。

### 5.2 2D マップの表示

- **入力**: 「ルートパス」（例: 現在ページのパス、または `/`）。
- **処理**: その直下の子パス（ページ一覧のうち `path` が 1 階層深いもの）を取得。子を「棚」として、1 行に 2 棚程度で奥に向かって並べる（Knowledge_Library の `shelf-row` 相当）。
- **スタイル**: 一点透視の廊下・床・棚。テーマ（fm26-blue）に合わせて青系・縁取りを適用。
- **インタラクション**: 棚クリック → そのパスをルートにした「本棚ビュー」を開く、または GROWI のそのパスを開く。

### 5.3 本棚ビューの表示

- **入力**: パス 1 つ（例: 現在ページのパス、または 2D マップで選んだ棚のパス）。
- **処理**: `path={そのパス}` でページ一覧を取得。直下の子をカード表示。
- **カード**: タイトル、サマリ（あれば）、子ページ数（あれば）。クリックで GROWI の該当ページへ遷移。
- **スタイル**: カードは角丸・青い縁（テーマ連携）。グリッド or フレックスで並べる。

### 5.4 パンくず・戻る

- 本棚ビューで「親パスに戻る」「ルートに戻る」を表示。2D マップと本棚の切り替えボタンも用意。

## 6. 技術構成

| 項目 | 方針 |
|------|------|
| **エントリ** | `client-entry.tsx`（React 使用） |
| **状態** | 現在の「ルートパス」「表示モード（マップ / 本棚）」「読み込み中」を React state で保持。 |
| **API 呼び出し** | GROWI のベース URL は `window.location.origin` または growiFacade から取得。`fetch('/_api/v3/pages/list?path=...')`。 |
| **スタイル** | CSS モジュール or 単一 CSS。fm26-blue の変数（`--grw-fm26-border` 等）を参照して統一。 |
| **DOM 挿入** | サイドバーにボタンとパネル用の div を追加。React はその div に mount。 |

---

## 7. フェーズ案

### Phase 1: 最小構成（本棚ビューのみ）

- プラグイン骨格（activate / deactivate / pluginActivators）。
- サイドバーに「本棚」ボタンを追加。
- クリックでパネルを開き、「現在ページのパス」をルートに子ページ一覧を API で取得してカード表示。
- カードクリックで GROWI のそのページを開く。
- **成果物**: 本棚ビューのみ動くプラグイン。

### Phase 2: 2D マップ追加

- 同じパネル内に「マップ」タブ or 切り替えを追加。
- ルートパスの直下の子を「棚」として 2D マップ表示（一点透視の廊下＋棚）。
- 棚クリックでそのパスをルートにした本棚ビューを表示（またはそのパスへ遷移）。
- **成果物**: 2D マップと本棚の両方。

### Phase 3: 体験向上

- パンくず・戻る・「ルートを変更」の入力欄。
- 読み込み中・エラー表示の整備。
- テーマ（fm26-blue）に合わせたスタイル調整。
- 必要なら「ルートパス」を URL クエリで指定できるようにする。

### Phase 4（任意）: 拡張

- 棚の「分野」をタグや親パス以外のルールで分類するオプション。
- キーボードショートカット（例: `Ctrl+Shift+B` で本棚パネル開閉）。

### Phase 5: Knowledge Graph・リンクプレビュー（bookshelf-map で実装）

- **グラフタブ**: パネルに「グラフ」タブを追加し、現在ページとリンク先・リンク元を 2D グラフで表示。
- **リンクのマウスオーバー小窓**: 自サイトリンクにホバーでリンク先ページの要約をポップオーバー表示。

※**コート／ボード配置 UI** は別プラグインとして検討。詳細は [Knowledge Graph・リンクプレビュー・コート配置UIの検討](./Knowledge%20Graph・リンクプレビュー・コート配置UIの検討.md) を参照。

## 9. リスク・確認事項

| 項目 | 内容 |
|------|------|
| **API の認証** | ブラウザから `/_api/v3/pages/list` を叩いたときに、GROWI のセッションで認証されるか。要確認。 |
| **CORS** | 同一オリジンであれば問題なし。GROWI.cloud でも同一オリジン想定。 |
| **ページ数制限** | list API の limit。子が大量の場合は「もっと見る」やページングを検討。 |
| **パスの取得** | 現在ページのパスを `growiFacade` や DOM からどう取るか。GROWI のクライアント API を確認。 |
