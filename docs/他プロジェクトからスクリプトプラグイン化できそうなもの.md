# 他プロジェクトの内容でスクリプトプラグインにできそうなもの

以下のフォルダを確認し、GROWI の**スクリプトプラグイン**（クライアント側の拡張）として実現できそうな機能をリスト化した。

- `I:\Projects\WEBUI_Dev`
- `I:\Projects\WEB_Tool`
- `I:\Projects\NDLOCR-Lite`
- `I:\Projects\Knowledge_Library`
- `I:\Projects\Dev_tool`

---

## 1. Knowledge_Library 由来

| 機能 | 元の内容 | プラグイン化のイメージ | 備考 |
|------|-----------|------------------------|------|
| **リンク・ページのグラフ表示** | 本・本棚・グラフ可視化（GROWI API 連携） | 現在ページから参照されているリンク／参照しているリンクをノードにして、ページ内 or サイドパネルでグラフ表示。GROWI API をフロントから呼ぶ | 既存の graph/explore の UI を GROWI 上に載せる |
| **2D マップ風ナビ** | explore-2d（図書館マップ、棚をクリックで遷移） | サイドバーや専用タブで「現在パス配下の子ページ」を 2D の棚・マップ風に並べ、クリックでそのページへ遷移 | ツリーの可視化の一種 |
| **本棚ビュー** | 本棚・本の一覧 | あるパスを「本棚」として、配下のページをカード一覧表示するビューをプラグインで追加 | 一覧の見せ方の拡張 |
| **Knowledge Schema（YAML フロントマター）の表示** | Core Concepts の id / level / depends_on / tags 等 | ページのメタデータ（YAML）をパースして、依存関係・タグ・レベルをサイドバーやバッジで表示 | メタデータの可視化 |

## 2. WEB_Tool 由来

| 機能 | 元の内容 | プラグイン化のイメージ | 備考 |
|------|-----------|------------------------|------|
| **ノード・エッジのグラフ表示** | nodes/edges API、3D ビュー | GROWI のページをノード、リンクをエッジとしてグラフ表示（2D で十分）。API でページ一覧・リンクを取得する必要あり | サーバーは GROWI 既存 API で代替可能 |
| **Markdown から構造を抽出して表示** | import_md（見出し→Process/Org/Tech、フェーズ） | ページ本文の見出しを解析し、「 Process / Org / Tech 」や「フェーズ」ごとにブロック表示 or 目次としてハイライト | クライアントで Markdown パースのみ |
| **3D ビュー** | view3d | ページツリーを 3D 空間に配置してナビする UI。負荷・実装コストは大きいが、コンセプトはプラグインで再現可能（Three.js 等） | 優先度低めで検討 |

## 3. Dev_tool 由来

| 機能 | 元の内容 | プラグイン化のイメージ | 備考 |
|------|-----------|------------------------|------|
| **BOM 風テーブルの表示・編集** | BomView（部品番号・名称・数量・仕様…） | 特定の Markdown テーブル or コードブロック（例: ```bom）を「BOM ビュー」として表示。GROWI の編集は既存のまま、表示だけリッチに | テーブル拡張 |
| **FMEA 風テーブルの表示** | FmeaView（機能・故障モード・影響・S,O,D,RPN・対策） | 同様に Markdown テーブル or 独自記法を「FMEA ビュー」として表示。RPN の計算はクライアントで可能 | テーブル拡張 |
| **タスク一覧ビュー** | TasksView | ページ内のチェックリストや特定記法のタスクを抽出し、一覧・進捗表示 | 既存のチェックリスト拡張 |
| **変数展開プレビュー** | variables / expandVariables | プレビュー時に `{{変数名}}` を展開して表示するオプション（変数定義は同一ページ or 別ページの YAML 等） | マークダウン拡張 |
| **スプリットビュー・テーマ切替** | Bokuchi 風の編集/プレビュー・テーマ | 編集画面のレイアウトやテーマは GROWI 本体の範囲なので、プラグインでは「プレビュー領域のスタイル調整」「フォントサイズ切替」程度が現実的 | 既存テーマプラグインと併用 |

## 4. WEBUI_Dev 由来

| 機能 | 元の内容 | プラグイン化のイメージ | 備考 |
|------|-----------|------------------------|------|
| **UI コンポーネントのスタイルガイド** | UI Lab | GROWI の「サンドボックス用ページ」で、ボタン・カード・フォーム等のスタイルを一覧表示するブロックを追加 | デザイン確認用 |
| **Cursor ルールの参照** | ルール MD 群 | プラグインというより GROWI に「ルール用ページ」を置いて参照する運用で十分。必要なら「特定パスをサイドバーに出す」程度 | 優先度低 |

## 5. NDLOCR-Lite 由来

| 機能 | 元の内容 | プラグイン化のイメージ | 備考 |
|------|-----------|------------------------|------|
| **OCR 結果の表示** | 国図 OCR（Python/ONNX） | OCR 自体はブラウザで再実装するのは重い。**「既に取得した OCR 結果テキストをページに貼る／表示を整える」**程度ならプラグインで可能 | 連携は「別ツールで OCR → テキストを GROWI に貼る」が現実的 |
| **画像のキャプション・構造化メタ** | リソースの XML / 座標 | 画像にメタデータ（領域・キャプション）を付与して表示する UI は、メタを JSON 等でページに持てばプラグインで表示拡張可能 | ニーズがあれば検討 |

---

## 優先度の目安（スクリプトプラグインとして）

| 順位 | 候補 | 理由 |
|------|------|------|
| 1 | **リンクのグラフ表示**（Knowledge_Library） | GROWI の「つながり」を可視化する需要が高く、API だけで実装可能 |
| 2 | **BOM / FMEA 風テーブル表示**（Dev_tool） | 既存 Markdown を活かしつつ、表示だけリッチにできる |
| 3 | **2D マップ風・本棚ビュー**（Knowledge_Library） | ツリーの別表現として、ナビ・探索の体験向上 |
| 4 | **Markdown 見出しの構造抽出表示**（WEB_Tool） | 長文の構成把握に役立つ。クライアントのみで実装可能 |
| 5 | **Knowledge Schema（YAML）の表示**（Knowledge_Library） | メタデータを扱う運用をしている場合に有効 |

---

## スクリプトプラグインで「やりにくい」もの

- **OCR エンジンそのもの**（NDLOCR-Lite）… ブラウザで ONNX を動かすのは重い。結果の表示・整形なら可。
- **ファイルシステム直叩き**（Dev_tool の開く/保存）… スクリプトプラグインはブラウザ内のため、GROWI の「ページ保存」経由になる。
- **常時動くバックエンド**（WEB_Tool / Knowledge_Library の FastAPI）… プラグインはクライアント拡張。バックエンドは GROWI 本体 or 別サービス＋API 連携で検討。
