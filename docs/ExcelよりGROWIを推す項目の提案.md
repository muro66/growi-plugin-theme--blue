# Excel より GROWI の機能を使った方がよいもの — 提案

BOM・FMEA・QFD および先に挙げたリストを前提に、「Excel ではなく GROWI でやった方がよい」ものを整理した。

---

## GROWI でやるメリット（共通）

- **リンク・参照** … 部品ページ・故障モードページ・要求仕様ページへ直接リンクでき、**単一の正（Single Source of Truth）** にできる
- **更新履歴・差分** … 誰がいつ変更したかが残り、**変更理由の追跡**がしやすい
- **検索** … 全文検索で「この部品番号が出てくるページ」をすぐ出せる
- **権限・共有** … チームで同じ場所を見て編集できる。Excel の「ファイル共有・ロック」より運用しやすい
- **テンプレート** … テンプレートプラグインで BOM/FMEA/QFD のひな形を配布できる
- **つながりの可視化** … スクリプトプラグインで**リンクのグラフ表示**をすれば、要求→機能→部品の関係を一覧できる

---

## 推奨：GROWI を主に使うとよいもの

### 1. BOM（部品表）

| 観点 | 提案 |
|------|------|
| **なぜ GROWI が向くか** | 部品ごとに**1ページ**を切り、仕様・図面リンク・変更履歴を一元化できる。親子の構成は「親ページ」に Markdown テーブルで BOM を書き、部品名を**子ページへのリンク**にすると、クリックで詳細に飛べる。 |
| **Excel との使い分け** | マスタとしての BOM は GROWI のページ＋テーブル。**コスト集計・発注用の CSV 出力**が必要なら、GROWI のテーブルをコピーするか、スクリプトプラグインで「BOM 風表示＋Export」を用意する。 |
| **プラグイン** | **BOM 風テーブル表示**（Markdown テーブルをリッチ表示）を入れると、Excel っぽい見た目のまま GROWI 上で編集・リンクを活かせる。 |

**結論**: **BOM は GROWI を主軸**にし、必要に応じて Excel/CSV は「エクスポート先」にするのがよい。

---

### 2. FMEA（故障モード影響解析）

| 観点 | 提案 |
|------|------|
| **なぜ GROWI が向くか** | 機能・故障モード・対策を**ページ**にし、FMEA 表は「それらへのリンク付きテーブル」にできる。RPN の計算は Markdown の数値で持ち、**スクリプトプラグインで FMEA 風表示＋RPN 自動計算**にすると Excel に近い操作性になる。 |
| **Excel との使い分け** | 作成・更新・レビューは GROWI。**監査用のスナップショット**が必要なら、その時点の表を CSV エクスポート or 印刷で残す。 |
| **プラグイン** | **FMEA 風テーブル表示**（S,O,D → RPN 表示・並び替え）があれば、Excel でやっていた見た目・運用を GROWI に寄せやすい。 |

**結論**: **FMEA も GROWI を主軸**にし、必要ならプラグインで「表の見た目・計算」を補う。

---

### 3. QFD（品質機能展開・品質の家）

| 観点 | 提案 |
|------|------|
| **なぜ GROWI が向くか** | 要求品質・技術要件・相関マトリクスを**1ページ or 関連ページ群**で管理できる。「要求」ページと「技術要件」ページをリンクでつなぎ、**リンクのグラフ表示**で「要求→技術」の関係を可視化できる。 |
| **Excel との使い分け** | 家の形のマトリクスそのものは Excel の方が描きやすい。GROWI では **「左壁＝要求一覧」「天井＝技術要件一覧」「部屋＝関係・重み」** を Markdown テーブル＋リンクで表現し、**見出しでセクション分け**するとよい。複数 QFD を同じプロジェクト配下のページで管理できる。 |
| **プラグイン** | 必須ではない。必要なら **Markdown 見出しの構造抽出表示**で QFD の章立てをサムネイル的に表示する程度でよい。 |

**結論**: **QFD の「中身・リンク・履歴」は GROWI**。描画の細かさが必要なら Excel で図を作り、GROWI に**図を貼る or リンク**で参照する運用もあり。

---

### 4. その他リストから「GROWI を優先したい」もの

| 項目 | 理由 |
|------|------|
| **リンクのグラフ表示** | 要求↔機能↔部品↔FMEA の**つながり**を可視化できる。Excel では難しい。 |
| **2D マップ・本棚ビュー** | プロジェクト・製品の**構成**を「棚」として見せられる。ナビ・探索に強い。 |
| **Knowledge Schema（YAML）の表示** | 知識の id・level・depends_on を GROWI のメタデータで持ち、**依存関係をバッジ・一覧で表示**できる。Excel の「参照」より追いやすい。 |
| **Markdown 見出しの構造抽出** | 長い設計書・手順書の**構成**を一目で把握できる。Excel のアウトラインに近いが、本文と一体。 |
| **タスク一覧ビュー** | チェックリストを GROWI に書き、プラグインで**一覧・進捗**表示。ToDo の共有・履歴が Excel より扱いやすい。 |

---

## Excel のままがよい／併用がよいもの

| 項目 | 理由 |
|------|------|
| **細かい数値計算・グラフ** | 多段計算・ソルバー・複雑なグラフは Excel が得意。GROWI には「結果の要約・リンク先」を書く。 |
| **定型の帳票・印刷レイアウト** | 発注書・検査表など**レイアウト固定の帳票**は Excel で作り、必要なら GROWI に「ひな形のリンク」を置く。 |
| **監査用のスナップショット** | ある時点の BOM/FMEA を**そのままの表で残す**必要がある場合は、CSV エクスポート or Excel に貼って保存。 |

---

## まとめ（優先して GROWI に寄せる順）

1. **BOM** … 部品をページ化し、リンク付きテーブル＋（必要なら）BOM 風表示プラグイン
2. **FMEA** … 機能・故障モードをページ化し、FMEA 風表示プラグインで RPN ・一覧を補助
3. **要求・仕様・設計の「つながり」** … リンクのグラフ表示プラグインで可視化
4. **QFD** … 要求・技術要件をページ＋テーブルで管理し、必要に応じて Excel は「図・帳票」用に併用
5. **タスク・チェックリスト** … GROWI のチェックリスト＋タスク一覧ビュー（プラグイン）で共有・進捗管理

これらを GROWI 中心にすると、**参照・履歴・検索・権限**を活かしつつ、Excel は「計算・帳票・スナップショット」用に残す形にしやすい。
