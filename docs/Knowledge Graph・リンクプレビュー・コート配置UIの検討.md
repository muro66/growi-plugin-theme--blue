# Knowledge Graph・リンクプレビュー・コート配置 UI — 検討メモ

①Knowledge Graph の効果的な表示、②リンク先のマウスオーバー小窓、③Football_Player のような「選手をコートに配置する」UI について整理する。

**方針**  
- **①・②**: [growi-plugin-bookshelf-map](https://github.com/muro66/growi-plugin-bookshelf-map) を発展させて実装（本棚パネルにグラフタブ追加、リンクプレビュー機能を同梱）。  
- **③ コート配置 UI**: 別途ブラッシュアップし、**独立プラグイン**として検討する。

---

## ① Knowledge Graph を GROWI で効果的に表示する方法

### 前提

- **ノード** = GROWI のページ
- **エッジ** = ページ間のリンク（`/page/...` の参照）
- データは GROWI API（`/api/v3/pages/list`、ページ本文からのリンク抽出）で取得可能

### 表示方法の候補


| 方法                     | 説明                                                    | 実装のしやすさ                  | 備考                          |
| ---------------------- | ----------------------------------------------------- | ------------------------ | --------------------------- |
| **A. 専用ページでグラフ表示**     | 1 ページに「このページを中心にしたグラフ」を 1 つ表示。ノードクリックでそのページへ遷移。       | ◎ スクリプトプラグインで実装可能        | 本棚プラグインと同じく、パネル or 専用ビューで描画 |
| **B. サイドパネルでミニグラフ**    | 現在ページと直近のリンク先だけを小さなグラフで表示。ドリルダウンは「ノードクリックでそのページを開く」。  | ◎ 同上                     | 本棚プラグインの「パネル」にグラフタブを追加する形   |
| **C. ページ本文中のリンクをグラフ化** | ページ内の `[/path](path)` をパースし、それらをノードにしたミニグラフをページ下部に表示。 | ○ リンク抽出はクライアントで可能        | 表示範囲を「このページ＋1 ホップ」にすると軽い    |
| **D. サイト全体の大きなグラフ**    | 全ページ・全リンクを取得して 1 つのグラフに。                              | △ API の limit・負荷・初期表示が重い | オプションで「範囲をパスで限定」すると現実的      |


### 推奨（bookshelf-map 発展）

- **Phase 2.5 として「グラフタブ」を追加**  
  - 本棚パネル内に「本棚」「マップ」「**グラフ**」のタブを設ける。
  - グラフタブでは「現在パス配下」または「現在ページ＋そのページが参照しているリンク＋参照されているリンク」をノードにし、**Cytoscape.js / vis-network / sigma.js** などで 2D グラフ描画。
- **データ取得**  
  - 現在ページの path を取得 → `/_api/v3/pages/list?path=...` で子一覧。  
  - リンク関係は「ページ本文を取得して Markdown の `](/)` を正規表現で抽出」するか、GROWI にリンク一覧 API があれば利用。
- **効果的な見せ方**  
  - 中心に「現在ページ」を置き、周囲にリンク先・リンク元を配置。  
  - ノードラベルはページタイトル（長い場合は省略）。  
  - クリックでそのページへ遷移。ズーム・パンはライブラリ任せ。

## ② リンク先の小窓をマウスオーバーで表示 — 可能か

### 結論：**可能**（スクリプトプラグインで実装できる）

### 仕組み

- GROWI 内の**自サイトリンク**（`/page/...` または相対パス）にマウスオーバーしたとき、
  - 一定時間（例: 500ms）後に **fetch** で該当ページの要約を取得（`/_api/v3/page` で body の先頭 N 文字など）。
  - 取得結果を **ポップオーバー（小窓）** で表示。タイトル・要約・サムネイル（あれば）を表示し、クリックでそのページへ遷移。

### 実装のポイント


| 項目        | 方針                                                                                   |
| --------- | ------------------------------------------------------------------------------------ |
| **対象リンク** | `a[href^="/page/"]` や GROWI のページリンクのみに限定（外部リンクは触らない）                                 |
| **トリガー**  | `mouseenter` でタイマー開始、`mouseleave` でキャンセル・小窓を消す                                       |
| **API**   | `GET /_api/v3/page?path=...` でページ取得。レスポンスの `body` や `revision.body` の先頭 200 文字程度を要約に |
| **表示**    | `position: fixed` の div をマウス位置の近くに表示。矢印は任意                                           |
| **キャッシュ** | 同じ path は一定時間キャッシュして API 呼び出しを削減                                                     |


### bookshelf-map との関係

- **別プラグイン**として「リンクプレビュー」だけ実装するのが分かりやすい。
- あるいは bookshelf-map に「リンクプレビュー機能」をオプションで含め、有効時だけ `a[href^="/page/"]` にマウスオーバーで小窓を表示する、という拡張も可能。

## ③ コート配置 UI — 別プラグインとして検討

### 方針

- **独立プラグイン**として検討する。bookshelf-map とは別リポジトリ・別ドキュメントでブラッシュアップする。

### 想定する機能（ツリー構築 UI）

- **右側**: **テンプレートをカード形式で一覧表示**。テンプレートとは「記事のひな形」や「よく使う構成」を指す想定。
- **ピックアップ**: カードを選択（クリック or ドラッグ）して、編集対象として取り込む。
- **名前付け**: ピックアップした項目にタイトルや識別名を付ける。
- **階層配置**: ドラッグ＆ドロップやインデント操作で、**親子・兄弟の関係**を UI 上で配置する。
- **記事ツリーの構築**: 上記で作った「ツリー構造」を、**各記事のページツリー（GROWI のパス階層）として反映**する。  
  - 例: ルート記事の下に「1. 概要」「2. 詳細」… のような子ページを、UI で並べてから一括でページ／パスを作成する。

### 参考にしたイメージ（Football_Player）

- 左: リスト、中央: コート（配置エリア）、右: 詳細 — のレイアウト。
- 本プラグインでは「右にテンプレートカード」「中央〜左にツリー編集エリア」のような配置で、**ツリー構造を UI で組み上げ、それを GROWI の記事階層として保存する**機能を想定。

### 技術メモ（別プラグイン検討時用）

- データ: テンプレート一覧の取得元（固定 YAML／専用ページ／API）、ツリー構造の保存（ページ本文のコードブロック JSON または GROWI のページ作成 API で実体化）。
- UI: ドラッグ＆ドロップ、インデント、リネーム、プレビュー。既存の「コートに配置」は「位置 (x,y)」だったが、本件は「階層（親子・順序）」が主。

## まとめ：bookshelf-map で実装するもの（①・②）

| 順序 | 内容 | 狙い |
|------|------|------|
| 1 | **② リンクのマウスオーバー小窓** | 全ページで自サイトリンクにホバーすると要約をポップオーバー表示。 |

**③ コート配置 UI（ツリー構築 UI）** は別プラグインとして検討。右にテンプレートカードを並べ、ピックアップ・名前付け・階層配置で記事のツリー構造を UI で構築し、GROWI のページ階層として反映する機能を想定。

## メモ：リンクのデータ取得（一旦パス）

- 本棚・グラフの表示で「リンクがおかしい」「期待する階層・一覧にならない」といった事象があり、**リンク／リストのデータ取得まわりが原因の可能性が高い**。
- API の仕様（`/_api/v3/pages/list` の返却内容・ソート・直下の子の取り方）、パスのエンコード/デコード、レスポンスの `path` の形式など、要検証。
- 対応は一旦パスし、上記をメモとして残す。必要になったら GROWI の API 仕様と実装を突き合わせて調査する。

## 技術メモ（グラフ・プレビュー・ドラッグ）

- **グラフ描画**: [Cytoscape.js](https://js.cytooscape.org/) / [vis-network](https://visjs.github.io/vis-network/docs/network/) / [sigma.js](https://www.sigmajs.org/) のいずれかを script プラグインから読み込み。
- **リンクプレビュー**: `document.querySelectorAll('a[href^="/page/"]')` に `mouseenter` / `mouseleave` を付け、fetch で `/_api/v3/page` を呼ぶ。
- **ドラッグ＆ドロップ**: HTML5 Drag and Drop API（Football_Player と同様）。配置座標はピッチを 0〜1 の正規化座標で持つとレスポンシブしやすい。
