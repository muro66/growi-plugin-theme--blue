# プラン：ツリー構築 UI プラグイン（カードで表現する記事ツリー）

GROWI の記事（ページ）ツリーを、**カードで表現したツリー構造**として編集し、テンプレートのピックアップ・名前付け・階層配置から実際のページ階層までを一貫した UI で構築するスクリプトプラグインの実装計画。  
「ツリー構造のカードでの表現」にこだわり、見た目と操作の両方で階層が直感的に分かるようにする。

---

## 1. 目標とコンセプト

### 1.1 目標

- **カードでツリーを描く**: ツリーの各ノードを 1 枚のカードで表し、親子・兄弟・深さがひと目で分かるようにする。
- **右にテンプレート、左にツリー**: 右側でテンプレートをカード一覧で選び、左（中央）のツリーエリアに配置・編集する。
- **GROWI のページ階層として保存**: 編集したツリーを、GROWI のパス階層（親ページ・子ページ）として作成・更新する。

### 1.2 ツリー構造の「カードでの表現」にこだわる点

| 観点 | 方針 |
|------|------|
| **1 ノード = 1 カード** | ツリーの各ノードを必ずカードとして表示する。プレーンなリストではなく、カードの枠・余白・影で「単位」を明確にする。 |
| **階層の見た目** | 深さに応じてカードを**インデント**し、必要なら**左ボーダー／背景の段階的な色**で階層を表現する。親カードの直下に子カードがぶら下がる形で、フォルダツリーのような視覚を目指す。 |
| **親子のつながり** | オプションで、親カードと子カードの間に**縦線（コネクタ）**を引く。カードだけでも階層は伝わるが、線があると「どのカードの子か」がより明確になる。 |
| **折りたたみ** | 子を持つカードは**開閉可能**にし、閉じているときはカードに「子が N 件」などのサマリを表示。ツリーが大きくなっても見通しを保つ。 |
| **ドラッグでの階層変更** | カードをドラッグして**別のカードの上／間／子としてドロップ**できるようにする。ドロップ位置で「兄弟として挿入」「子の先頭として追加」を区別する。 |
| **カード上の情報** | 各カードに**タイトル（編集可）**、必要なら**パスプレビュー**や**テンプレート種別のバッジ**を載せ、何のノードかが分かるようにする。 |

## 2. UI レイアウト

```
+------------------+------------------------------------------+------------------+
|                  |                                          |  テンプレート     |
|  （オプション）   |         ツリー編集エリア                  |  カード一覧      |
|  現在のページ     |         （カードで表現したツリー）         |  ┌────┐ ┌────┐  |
|  パス／保存等     |  ┌─────────────────────────────┐         |  │ T1  │ │ T2  │  |
|                  |  │ ルート / 記事タイトル        │         |  └────┘ └────┘  |
|                  |  └─────────────────────────────┘         |  ┌────┐         |
|                  |    ├─ ┌─────────────────────┐             |  │ T3  │ ...    |
|                  |    │  │ 1. 概要              │             |  └────┘         |
|                  |    │  └─────────────────────┘             |                  |
|                  |    ├─ ┌─────────────────────┐             |  クリック or     |
|                  |    │  │ 2. 詳細              │             |  ドラッグで      |
|                  |    │  │   ├─ ┌───────────┐   │             |  ツリーに追加    |
|                  |    │  │   │  │ 2.1 仕様  │   │             |                  |
|                  |    │  │   │  └───────────┘   │             |                  |
|                  |    │  │   └─ ...              │             |                  |
|                  |    │  └─────────────────────┘             |                  |
|                  |    └─ ...                                   |                  |
+------------------+------------------------------------------+------------------+
```

- **左**: 現在編集対象のルートパス、保存・キャンセル、必要なら「GROWI から読み込み」などの操作。
- **中央**: ツリー編集エリア。**カードの縦並び＋インデント＋コネクタ**でツリーを表現。
- **右**: テンプレートをカード形式で並べ、クリックまたはドラッグでツリーに追加。

## 3. カードの仕様（ツリーノード用）

### 3.1 見た目

- **形状**: 角丸の矩形カード。影または枠線で他エリアと区別。
- **内容**:  
  - タイトル（1 行、編集可能）  
  - オプション: パスプレビュー（保存前は「予定パス」）、テンプレート名バッジ、子の数
- **階層の表現**:
  - **インデント**: 深さ d に対して左マージン（または padding-left）を `d * インデント幅`（例: 24px）で増やす。
  - **左ボーダー**: 深さに応じた色（例: 1 段目は青、2 段目は緑）で縦線を引く、またはカード左端にアクセントを付ける。
  - **コネクタ**: 親のカード下端から子のカード上端へ縦線。兄弟間は縦線を共有してもよい（ツリービュー風）。

### 3.2 インタラクション

| 操作 | 動作 |
|------|------|
| **クリック** | カードを選択（編集モード: タイトル編集やプロパティ表示）。 |
| **ドラッグ** | カードを掴んで移動。ドロップ先で「兄弟の前後」または「子として追加」を判定。 |
| **インデント / アウトデント** | ボタンまたはショートカットで、選択カードを 1 段階子にする / 1 段階上に上げる。 |
| **開閉** | 子を持つカードの左端に三角アイコン。クリックで子を表示/非表示。 |
| **削除** | カード上の削除ボタンまたはキーで、そのノード（と子孫）を削除。 |

### 3.3 ドロップゾーン

- **カードの上半分にドロップ**: そのカードの**直前の兄弟**として挿入。
- **カードの下半分にドロップ**: そのカードの**直後の兄弟**として挿入。
- **カード中央または「子を追加」ゾーンにドロップ**: そのカードの**子の先頭**として追加。
- 子が折りたたまれている場合は、カード上に「ここにドロップで子として追加」のヒントを表示してもよい。

---

## 4. テンプレートカード（右パネル）

- **見た目**: ツリーノード用カードより小さめ・シンプル。テンプレート名・アイコン・短い説明（あれば）を表示。
- **操作**:  
  - **クリック**: ツリーの「選択中のノードの子」または「ルートの直下」に、そのテンプレートから生成した新カードを 1 枚追加。  
  - **ドラッグ**: ツリーエリアの任意のドロップゾーンに持っていって追加。
- **テンプレートの中身**: 初期は「見出し＋本文のひな形」を 1 ノードに対応させる想定。データは YAML/JSON または専用ページで定義。

---

## 5. データ構造

### 5.1 ツリーのノード（メモリ上）

```ts
interface TreeNode {
  id: string;           // 一意
  title: string;        // 表示・保存用タイトル
  path?: string;       // GROWI に保存済みの場合のパス
  templateId?: string; // 元テンプレート
  children: TreeNode[];
  order?: number;       // 兄弟内の並び順
}
```

### 5.2 テンプレート定義（例）

```yaml
# プラグイン設定 or 専用ページ
templates:
  - id: section
    name: セクション
    defaultTitle: "セクション"
    body: "## {{title}}\n\n"
  - id: overview
    name: 概要
    defaultTitle: "概要"
    body: "## 概要\n\n（ここに概要を書く）\n"
```

### 5.3 GROWI への保存

- **案 A**: 編集結果を JSON でルートページの本文（コードブロック）に書き、プラグインがその JSON を「設計図」として読み、必要に応じてページ作成 API で実体化する。
- **案 B**: ツリーをそのまま GROWI のページ作成 API で反映する（ルートの子を順に作成・更新）。ツリーと実ページを 1:1 で対応させる。

## 6. 実装フェーズ（カード表現を段階的に完成させる）

### Phase 1: カードでツリーを「表示」する

- **目的**: ツリー構造をカードで表現する UI を固める。
- **内容**:
  - 固定のツリーデータ（JSON）を読み込み、**カードの縦並び＋インデント**で表示。
  - 各ノードを 1 枚のカードとして描画（タイトル、左インデント、オプションで左ボーダー）。
  - 子は親の下にインデントして並べる。コネクタ（縦線）はオプションで実装。
- **成果物**: ツリー編集エリアのみ。パネル or 専用ページで「カードツリー」が表示できる。

### Phase 2: カードの選択・編集・並び替え

- **目的**: カード単位でツリーを編集できるようにする。
- **内容**:
  - カードの選択状態、タイトル編集（インライン or モーダル）。
  - **インデント / アウトデント**ボタン（またはキーボード）で階層を変更。
  - **ドラッグ＆ドロップ**で並び順と親子を変更。ドロップゾーン（上半分／下半分／子として）の判定。
  - 子を持つカードの**開閉**（折りたたみ）。
- **成果物**: メモリ上のツリーを、カード操作だけで編集できる。

### Phase 3: テンプレートパネルと「ピックアップ」

- **目的**: 右側からテンプレートを選んでツリーに追加する。
- **内容**:
  - 右パネルに**テンプレートカード**を一覧表示（データは固定 or 設定から読み込み）。
  - テンプレートのクリックで「ルート直下 or 選択ノードの子」に新規カードを追加。
  - テンプレートのドラッグでツリーエリアの任意の位置に追加。
  - 追加したカードに**名前付け**（タイトル編集）。テンプレートの defaultTitle を初期値に。
- **成果物**: テンプレートからツリーを組み立てられる。

### Phase 4: GROWI との連携

- **目的**: ツリーを GROWI のページ階層として保存・読み込みする。
- **内容**:
  - **保存**: ツリーを JSON にシリアライズし、ルートページの本文のコードブロックに書き込む（案 A）、またはページ作成 API で子ページを生成（案 B）。
  - **読み込み**: 現在ページの子を API で取得し、ツリー構造に変換してカード表示。既存記事のツリーをカードで見て編集できるようにする。
  - パスのプレビュー表示（保存後のパス or 予定パス）。
- **成果物**: GROWI の記事ツリーとカードツリーが往復できる。

### Phase 5: カード表現の仕上げ・UX

- **目的**: ツリーのカード表現をさらに分かりやすくする。
- **内容**:
  - コネクタ（親子を結ぶ縦線）の描画を整える。
  - 深さによる色分け、ホバー・選択状態の明確化。
  - キーボード操作（矢印で移動、Tab/Shift+Tab でインデント/アウトデント）、アクセシビリティの考慮。
  - 空状態・エラー表示・読み込み中の表示。
- **成果物**: ツリー構造のカードでの表現が一通り完成し、実用レベルになる。

## 7. 技術メモ

- **フレームワーク**: React が無難（既存の bookshelf-map と同様）。状態はツリー構造（`TreeNode[]` のルート配列 or 1 ルートノード）を state で保持。
- **ドラッグ**: HTML5 DnD または `@dnd-kit` 等のライブラリで「リスト内・階層付き」のドラッグを実装。ドロップゾーンの判定（上/下/子）を明確に。
- **コネクタ**: SVG を編集エリアの背面に配置し、各親子のカード位置から線を引く。または CSS の border-left + 擬似要素で簡易的に表現。
- **スタイル**: テーマ（fm26-blue 等）に合わせつつ、カードの影・角丸・インデント幅を統一し、「ツリーのカード」であることが分かるデザインにする。
